## 5. 데이터 수집 전략

### 5.1 데이터 소스 우선순위

#### 1순위: 공공기관 API (무료)

**나라장터 API**:
- 입찰공고 정보
- 낙찰결과 정보
- 무료 사용 가능
- 안정적 데이터 제공

**장점**:
- 무료
- 안정적
- 정확한 데이터
- 법적 문제 없음

**단점**:
- API 제한 (시간당 요청 수)
- 일부 데이터 부족 가능

**구현 난이도**: 낮음  
**비용**: 무료  
**예상 시간**: 1-2주

#### 2순위: 웹 스크래핑 (보완)

**대상 사이트**:
- 지자체 웹사이트
- 공공기관 웹사이트
- API로 제공되지 않는 정보

**장점**:
- 추가 데이터 확보
- 빠른 수집 가능

**단점**:
- 서버 부하
- 법적 리스크
- 유지보수 필요
- robots.txt 준수 필요

**구현 난이도**: 중간  
**비용**: 서버 리소스  
**예상 시간**: 2-4주

#### 3순위: 수동 수집 (특수 케이스)

**대상**:
- API/스크래핑 불가능한 정보
- 검증이 필요한 정보
- 특수 공고

**장점**:
- 정확한 데이터
- 특수 정보 확보

**단점**:
- 인력 필요
- 시간 소요
- 비용 발생

**구현 난이도**: 낮음  
**비용**: 인건비  
**예상 시간**: 지속적
